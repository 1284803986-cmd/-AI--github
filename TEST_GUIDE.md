# 小程序小白测试说明

这份说明是给完全不懂代码的人看的。你只需要按步骤打开、点击、测试，不需要改代码。

## 一、打开小程序

1. 打开电脑上的 **微信开发者工具**。
2. 选择 **导入项目** 或 **打开项目**。
3. 项目目录选择：

```text
E:\教育小程序\-AI--github\miniapp
```

4. 注意：不要选择 `dist` 文件夹。
5. 进入项目后，点击微信开发者工具上方的 **编译**。
6. 如果能看到首页，说明小程序前端已经打开成功。

## 二、首页怎么测试

首页主要看这几块：

- 左上角选择年级。
- 年级旁边选择上册 / 下册。
- 中间有：首页、语文、数学、英语。
- 首页里有：继续学习、今日学习概况、常用入口。

测试方法：

1. 切换年级和上册 / 下册，看页面是否正常。
2. 点击“语文、数学、英语”，看能不能切换。
3. 查看“继续学习”和“今日学习概况”是否显示正常。

如果还没有做过题，“继续学习”显示没有练习记录是正常的。

## 三、章节练习怎么测试

1. 首页点击 **章节练习**，或者底部点击 **练习**。
2. 选择年级、学科、上册 / 下册。
3. 进入章节列表。
4. 点击章节。
5. 点击题型。
6. 进入做题页面。

目前主要有内容的是：

```text
二年级 / 数学 / 下册
四年级 / 数学 / 下册
```

如果选择了没有导入题库的年级、学科或上下册，页面显示“暂无内容”是正常的，不是程序出错。

## 四、做题页面怎么测试

做题页面一次只显示一道题。

页面下方有三个按钮：

```text
上一题   提交答案   下一题
```

测试方法：

1. 输入答案或选择答案。
2. 点击 **提交答案**。
3. 答对后，一般会自动进入下一题。
4. 答错后，会停留在当前题，需要手动点 **下一题**。
5. 故意做错一道题，再去错题本检查是否加入。

## 五、错题本怎么测试

1. 底部点击 **错题**。
2. 查看刚才做错的题是否出现。
3. 看题目、你的答案、正确答案、解析是否显示。
4. 如果重新做对，错题可能会从错题本中移除或变成已掌握状态。

注意：错题本数据一般存在本机缓存里。如果清理缓存或换设备测试，错题本可能会清空，这是正常现象。

## 六、学生作业怎么测试

1. 首页点击 **学生作业**。
2. 输入老师提供的作业码。
3. 如果能进入作业页面，就继续完成题目。

注意：如果作业码、老师作业列表打不开，可能是后端服务没有启动，需要先启动后端。

## 七、拍照搜题怎么测试

1. 首页点击 **拍照搜题**。
2. 可以选择拍照上传，或者从相册上传。
3. 当前第一版还没有真正接入图片文字识别。
4. 推荐先使用 **手动输入题目**。
5. 输入题目后点击 **查看解析**。
6. 查看是否出现模拟解析。
7. 再点击 **生成同类题**，看是否出现同类题。

## 八、我的页面怎么测试

1. 底部点击 **我的**。
2. 查看是否能进入历史记录、关于我们、隐私说明等页面。
3. 历史记录放在“我的”页面里，不一定放在首页。

## 九、学习数据说明

首页的这些数据一般存在本机缓存里：

- 继续学习
- 今日学习概况
- 错题本

如果清理缓存、重新安装小程序、换电脑或换设备测试，这些数据可能会清空，这是正常现象。

## 十、推荐测试顺序

1. 打开微信开发者工具。
2. 选择 `miniapp` 目录。
3. 点击 **编译**。
4. 查看首页是否正常。
5. 进入章节练习。
6. 选择 `二年级 / 数学 / 下册` 或 `四年级 / 数学 / 下册`。
7. 选择章节。
8. 选择题型。
9. 做一道题。
10. 回首页，看“继续学习”和“今日学习概况”是否更新。
11. 故意做错一道题。
12. 去错题本，看错题是否出现。
13. 点击拍照搜题，测试手动输入和模拟解析。

## 一句话总结

这是一款给小学生用的学习练习小程序，核心是“按年级、学科、上下册、章节刷题”，做错的题自动进入错题本，首页显示学习进度，后期可以继续接入拍照搜题、AI解析、老师布置作业和学习数据统计。
